# 审计记忆与可交付阈值

## 一、固定喂料记忆（后续相关课程持续沿用）

以下问题已经在本轮测试中多次出现，后续 rewrite / audit / patch 阶段都应默认记住，避免重复踩坑：

1. 监管演进时间线
- 1992年10月：国务院证券委员会和中国证监会成立。
- 1998年11月：中国保监会成立。
- 1998年机构改革后：国务院证券委员会被撤销，证券监管职责进一步集中到中国证监会。
- 2003年4月：中国银监会成立。
- 写法重点：如果讲“分业监管演进”，应把证监会、保监会、银监会与人民银行监管职能分离过程连成线，而不是只点名某一个机构。

2. 《证券法》时间口径
- 1998年12月29日：通过/公布。
- 1999年7月1日：起施行。
- 若按课堂/教材口径压缩，可写成“1998年通过，1999年起施行”；不要把“通过”和“施行”混成一个点。

3. Shibor 报价行口径
- 课堂原始材料口径：最初 16 家报价行。
- 现行公开常见口径：18 家。
- 写法重点：正文应直接提示“做题先看题干背景年份/口径”，不要把两个口径糊成一个值。

4. 术语规范
- 统一写法：中央结算公司（中债登）。
- 统一写法：中国证券登记结算有限责任公司（中证登）。
- 避免出现：中正灯、中载登等转写/拼写错误。

5. 笔记体要求
- 删除“可概括为”“可形象理解为”“我们称之为”“大家知道”等课堂转述腔。
- 能用紧凑列表解决的，不滥用表格。
- 重点、易错、 高频要紧贴知识点本体，而不是另起说明区。

## 二、审计 PASS / REVISE 判定阈值

审计的目标是判断“是否已达到可交付标准”，不是追求绝对完美，也不是为了扮演审计员而每轮强行找新问题。

### 只有以下情况，才判 REVISE
- 存在会影响考试理解或答题的事实性错误。
- 漏掉关键知识点、关键时间点、关键机构关系、关键分类口径。
- 出现明显回复腔、客服腔、总结报告腔，已经偏离课程笔记交付形态。
- 重点/高频/易错体系严重缺失，导致无法直接复习使用。

### 以下情况，不应单独导致 REVISE
- 轻微措辞偏好差异。
- 可进一步优化但不影响交付的结构选择。
- 表格与列表之间的风格偏好问题。
- 已修复旧问题后，审计仅换个说法重复驳回。
- 可补充但不补也不影响当前交付的“锦上添花型”建议。

### 审计输出原则
- 如果稿件已经可直接交付，应判 PASS，并把剩余内容放到“可优化项”或“后续可继续优化”里，而不是继续判 REVISE。
- 不要为了显得严格而制造新的“疑似错误”。
- 对课堂材料与现行公开口径并存的情况，要先区分“课堂口径 / 教材口径 / 现行口径”，不能简单粗暴判错。
