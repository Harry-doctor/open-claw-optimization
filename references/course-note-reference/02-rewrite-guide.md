# 成稿重写阶段参考规约

你要把结构化底稿重写成“可直接复习的长期课程笔记”，并向参考文档的交付形态对齐。

必须继承的样式骨架：
1. 标题层级稳定：总标题 → 模块标题 → 知识点标题。
2. 每个知识点优先写“核心内容”，再把老师补充、考试提示、易错警示自然嵌入附近。
3. 标签体系固定且显眼：`【重点】`、`【高频考点】`、`【易错警示】`、`【了解/低频】`。
4. 重点和高频考点不能集中堆到文末，必须跟着具体知识点走。
5. 允许适度改成流程、时间轴、对照、关系式表达，但不要为了形式硬堆表格。
6. 语言要收束、稳定、像同一套课的长期笔记，不要像聊天回复、摘要、方案说明。
7. 对“不考/简单了解”内容只做降级处理，不要在该阶段扩写外部知识。
8. 如果样式选择发生冲突，优先贴近参考文档的呈现习惯，而不是自由发挥。

禁止项：
- 禁止出现“根据你提供”“如果你需要”“我可以继续帮你”等答复腔。
- 禁止把全文写成总结报告、学习建议、方法论文章。
- 禁止在文末新增“本章重点汇总/常考点总表/重要时间点总表”这类集中区。

从参考文档抽出的成稿风格：
- 句子短、信息密、概念准。
- 先把知识讲清，再贴重点与易错提醒。
- 老师课堂补充保留，但会被收束成适合复习的表达，而不是原话照录。
- 对高频混淆项，会直接在对应位置做区分提醒。

代表片段（只学交付形态，不要复述内容）：

## 片段A：概念解释后立刻贴重点
债券登记可以理解为：记录债券持有人是谁，以及持有人发生变化时如何更新。
主要包括三类：初始登记、变更登记、退出登记。
流程式抓手：初始登记 → 变更登记 → 退出登记
【重点】三类登记的定义和作用要对应起来。
【易错警示】“托管”“登记”“结算”不是完全同义概念。

## 片段B：历史脉络写法
所谓“大一统”，核心是当时金融体系高度集中统一管理。
其主要表现为：……
【重点】“唯一银行 + 双重职能”是理解大一统金融体系的抓手。
【易错警示】题目若把“大一统”说成多银行并存、分工明确，通常是错项。

## 片段C：板块/制度类写法
课堂提到的主要板块包括：主板、创业板、科创板、北交所、全国股转系统（新三板）等。
【重点】多层次资本市场考查时，通常围绕板块定位、服务对象、准入门槛来出题。
【易错警示】中小板已并入深市主板，现行记忆中不要再把中小板当作独立板块。
