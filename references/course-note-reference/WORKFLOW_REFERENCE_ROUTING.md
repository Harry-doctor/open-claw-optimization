# 课程笔记参考样例接线说明

参考文档：`https://rcnzx9z76i1x.feishu.cn/docx/YAmMdI3aDoIBLnxBiSCcZntWnjc`
标题：`课程1-中国的金融体系与债券市场`
用途：作为课程笔记工作流的长期参考样例。

## 四个稿件处理阶段如何喂

### 1. 结构提纯
- 模型：`gemini-3.1-flash-lite-preview`
- 喂什么：结构规约、标签体系、保真/降噪规则、少量代表片段
- 不喂什么：整篇参考全文逐字模仿
- 目的：让模型学会“怎么组织底稿”，不是学句子

### 2. 成稿重写
- 模型：`gpt-5.4`
- 喂什么：成稿风格规约、标签落点规则、章节组织方式、少量代表片段
- 不喂什么：无关全文冗长样例
- 目的：把底稿统一成与参考文档一致的长期课程笔记风格

### 3. 审计
- 模型：`gemini-3.1-flash-lite-preview`
- 喂什么：质检标准、样例风格要求、高优先级检查点
- 不喂什么：整篇参考正文
- 目的：节省 token，只让模型按参考样例的交付标准挑问题

### 4. 最终修补
- 模型：`gpt-5.4`
- 喂什么：修补标准、禁止项、标签体系、少量代表片段
- 不喂什么：重复的大段无效参考正文
- 目的：根据审计单，把稿件修回参考文档同级交付形态

## 为什么这样喂

- 结构提纯和审计更偏规则识别与检查，适合低成本 Gemini Lite。
- 成稿重写和最终修补更偏执行稳定性与细节收束，适合 GPT-5.4。
- 不把整篇参考文档在每个阶段反复塞进去，是为了长期控 token。
- 参考文档的价值主要在“标准与形态”，不是让每个阶段都全文照抄。
