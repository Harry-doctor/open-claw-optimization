# 最终修补阶段参考规约

这一阶段要把“当前最终稿 + 审计单”修成接近参考文档的最终交付形态。

修补原则：
1. 先修严重漏点、术语风险、时间点风险，再修风格与结构问题。
2. 修补后输出必须仍然是完整课程笔记正文，不得退化成问题清单或建议列表。
3. 继续沿用参考文档的标签体系：`【重点】`、`【高频考点】`、`【易错警示】`、`【了解/低频】`。
4. 重点与易错项继续贴着知识点本体修，不得集中搬到文末。
5. 如果审计指出“表达过泛/考试导向偏弱”，优先参考样例的写法：补足抓手、对照、时间轴、易错区分，而不是堆空话。
6. 对不确定内容保持谨慎，不要为了修补而凭空造知识点。
7. 修补完的文稿必须像同一套长期维护的课程笔记，不像二次回复。

禁止项：
- 禁止输出审计报告口吻。
- 禁止输出“已根据审计修改如下”之类过程说明。
- 禁止新增参考文档中没有的集中总结板块。

参考文档最值得对齐的地方：
- 标签落点准确
- 易错混淆项会被直接写在相关知识点附近
- 历史/制度/市场结构信息会被整理成便于记忆的抓手
- 全文语气稳定，不散、不虚、不像客服
